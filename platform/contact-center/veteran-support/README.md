# Veteran Support Knowledge Hub

## Overview
The Veteran Support Knowledge Hub provides centralized access to documentation, processes, and resources for engaging with the Veteran Support team.

This repository is designed to improve:
- Discoverability of documentation
- Consistency across workflows
- Self-service access for product and engineering teams

---

## 📍 Start Here

If you are unsure where to go, use the guide below:

- **Need to engage Veteran Support?**
  → See `01-getting-started/how-to-engage-veteran-support.md`

- **Looking for processes or workflows?**
  → See `02-processes-sops/`

- **Responding to an incident or escalation?**
  → See `03-playbooks/`

- **Need a template?**
  → See `04-templates/`

- **Looking for data or reporting guidance?**
  → See `05-data-insights/`

- **Training or onboarding materials?**
  → See `06-training-enablement/`

## 🧭 Source of Truth Guidelines

To maintain consistency:

- **GitHub (this repo)**  
  → Operational processes, templates, and structured documentation  

- **Platform Pages**  
  → External-facing guidance for product teams  

- **SharePoint**  
  → Internal collaboration and working documents  

---

## 📝 Contribution Guidelines

When adding or updating documentation:

1. Place content in the appropriate folder
2. Use clear, descriptive file names
3. Avoid duplicate documentation across platforms
4. Update existing documents when possible instead of creating new ones

---

## 🔁 Maintenance & Ownership

- Documentation should be reviewed periodically for accuracy
- Each document should have a clear owner (when applicable)
- Outdated content should be moved to `/08-archive/`
