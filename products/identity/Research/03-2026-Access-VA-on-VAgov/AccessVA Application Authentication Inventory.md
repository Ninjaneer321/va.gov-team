## AccessVA Application Inventory

The table below lists the unique applications currently surfaced through AccessVA.  
Each row represents a single application and captures the minimum metadata needed to evaluate how it may be supported through the Unified Sign-in Page (`/sign-in`).

### Fields

- **App name** — exact name displayed in AccessVA
- **Where found** — AccessVA audience category where the application was listed
- **Source link** — location where the application was discovered
- **Requires auth** — whether authentication appears required
- **LOA / IAL / AAL** — observed assurance requirement, if identifiable
- **Supported IdPs** — identity providers visible in the authentication flow (if known)
- **Redirect mechanism / params** — observed redirect behavior during authentication
- **Post-auth redirect URL** — destination reached after authentication
- **Sign-in Service eligible** — preliminary determination of whether the app could be supported by the VA.gov Sign-in Service
- **Notes** — observations, open questions, or screenshot references

Values may be marked **Unknown** where behavior could not be confirmed due to network restrictions, external login systems, or opaque redirect behavior.

This inventory supports Identity FE evaluation of **USiP onboarding** and potential future VA.gov deep-link routing (for example: `va.gov/application=`).
